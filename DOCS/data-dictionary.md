# Flow Control – Data Dictionary (Stage 0)

מסמך זה מרכז את תוצרי שלב 0 מהתוכנית: זיהוי מקרי השימוש העיקריים ומיפוי ישויות/שדות לפני שניגשים לסכמה בפועל.  
מקורות עיקריים שנקראו:  
- `DOCS/התכתבות Flow Control  10.11.25.txt` – תיאור מלא של הזרימות, הדשבורד וההתראות.  
- `DOCS/DOS from APP/…` (בעיקר קבצי *Entities* עם הפירוט הכתוב של every entity).  
- קובצי CSV לדוגמה (`DOCS/Reagent_2025-10-30.csv`, `DOCS/ReagentBatch_2025-10-30.csv`, `DOCS/Order_2025-10-30.csv`, וכו’) שנותנים מבנה נתונים אמיתי.  

> **הערה:** כאשר שם תיקייה מכיל תווים משובשים, אפשר להשתמש בשמות הקצרים (8.3) ש־Windows מייצרת, לדוגמה: `DOCS\DOS from APP\_ENTIT~1\Reagent_....md`.

## 3. פערים / החלטות נדרשות בהמשך
1. **שדות Enums** – יש להחליט אילו ערכי enum נכנסים לסכמה (לדוגמה `ReagentBatch.status`, `Order.status`, `Alert.severity`). ניתן לייבא ישירות מהקבצים לעיל.  
2. **קישור למסמכים** – בכל ישות שיש בה `documents[]` (Delivery, Shipment, COA) צריך לבחור פתרון קבצים (S3/Azure) ולהחזיק URL חתום.  
3. **אבטחת נתונים** – יש להוסיף `created_by`, `updated_by`, `hospital_scope` לרוב הישויות למקרה של עבודה מול כמה בנקי דם (כפי שנרמז בצ’אט).  
4. **מיפוי SAP** – יש לוודא האם מזהי SAP (permanent order / PO) צריכים טבלת Staging נפרדת או שדות ב־Order. כרגע טבלה בנויה על בסיס ההמלצה בקובץ ה־Entity.
