# אפיון תפקודי - CriticalActions

## מטרה
הצגת רשימת פעולות קריטיות הדורשות טיפול מיידי במרכז הבקרה.

## תיאור למשתמש
בראש מרכז הבקרה - רשימת "המלצות ופעולות קריטיות":
- 🚨 סדר לפי דחיפות (critical → high → medium)
- 💡 כל פעולה עם הסבר ברור
- 🔗 לחיצה → מעבר למסך הרלוונטי
- ✅ כשאין פעולות → "הכל מעודכן!"

## סוגי פעולות

### Critical (אדום):
- פג תוקף היום
- אזל מלאי מוחלט
- תקלת מערכת

### High (כתום):
- ספירת מלאי > 45 יום
- הזמנה ללא פרטי SAP > 7 ימים
- פג תוקף בעוד < 7 ימים

### Medium (כחול):
- מלאי נמוך (< 2 חודשים)
- משיכה ממתינה > 3 ימים
- COA חסר

## מקרי שימוש

### UC1: פעולה קריטית
1. נכנס לדשבורד
2. רואה כרטיס אדום: "דרוש טיפול מיידי ב-3 ריאגנטים שפג תוקפם"
3. לוחץ
4. עובר לדף ניהול פגי תוקף
5. רואה את 3 הריאגנטים
6. מטפל בהם
7. חוזר לדשבורד → הכרטיס נעלם

### UC2: הכל תקין
1. נכנס לדשבורד
2. רואה: "הכל מעודכן! אין פעולות קריטיות"
3. 💚 רקע ירוק
4. סימן ✓
5. רגוע

## כללי עסקיים

1. **Maximum 5 actions** - רק 5 הראשונות
2. **Priority sorting** - תמיד critical → high → medium
3. **Auto-refresh** - כל 5 דקות
4. **Action tracking** - מתי נלחץ, מתי טופל

## UI Elements

- Card header: 💡 + "המלצות ופעולות קריטיות"
- Action items: צבע + אייקון ⚠️ + טקסט + חץ ◀
- Empty state: ✅ + "הכל מעודכן"
- Smooth transitions בין actions
