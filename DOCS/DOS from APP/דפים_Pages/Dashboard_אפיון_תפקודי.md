# אפיון תפקודי - דף מרכז הבקרה (Dashboard)

## 1. מטרה ומיקוד

### 1.1 מטרת הדף
דף מרכז הבקרה משמש כנקודת מוצא מרכזית למערכת ניהול המלאי של בנק הדם. הדף מספק סקירה כוללת של מצב המלאי, התראות קריטיות, והחזרות מהירות לפעולות עיקריות.

### 1.2 קהל יעד
- **משתמשים עיקריים**: טכנאים וצוות מעבדה שמבצעים סריקת בוקר יומית
- **משתמשי מנהל**: מנהלי מעבדה שעוקבים אחרי מצב מלאי כללי
- **מנהלי מערכת**: צוות IT שמנטר את תקינות המערכת

## 2. מקרי שימוש עיקריים

### UC-001: סקירת מצב בוקר
**שחקן**: טכנאי מעבדה  
**תנאי מקדים**: המשתמש התחבר למערכת  
**זרימה**:
1. משתמש נכנס למרכז הבקרה
2. המערכת מציגה כרטיסי סיכום עם מספרים מעודכנים
3. משתמש סורק את קטע "פעולות קריטיות"
4. משתמש לוחץ על פעולה דחופה (אם קיימת)
5. המערכת מנתבת לדף הרלוונטי

**תוצאה**: משתמש מודע למצב המלאי והפעולות הנדרשות

### UC-002: טיפול בהתראה דחופה
**שחקן**: טכנאי/מנהל  
**תנאי מקדים**: קיימת התראה קריטית (פגי תוקף/מלאי נמוך)  
**זרימה**:
1. משתמש רואה קטע "פעולות קריטיות" בראש הדף
2. משתמש מזהה התראה באדום (קריטי) או כתום (דחוף)
3. משתמש לוחץ על ההתראה
4. המערכת מעבירה לדף הטיפול המתאים

**תוצאה**: משתמש מטפל בבעיה בזמן

### UC-003: ניווט מהיר לפעולה נפוצה
**שחקן**: כל משתמש  
**זרימה**:
1. משתמש רואה את קטע "ניווט מהיר" בצד
2. משתמש מרחיב קטגוריה
3. משתמש בוחר פעולה
4. המערכת מנתבת לדף המבוקש

**תוצאה**: גישה מהירה לפעולות

## 3. כללי עסקים וחישובים

### 3.1 ריאגנטים בתפוגה קרובה
**הגדרה**: תפוגה תוך 14 יום, טרם תועד, טרם פג

### 3.2 ריאגנטים במלאי נמוך
**הגדרה**: פחות מחודש מלאי או פחות מ-5 יחידות

### 3.3 פעולות קריטיות
- פג תוקף היום → **קריטי**
- ספירת מלאי 45+ ימים → **קריטי**
- ספירת מלאי 30-44 ימים → **גבוה**
- מלאי נמוך → **בינוני**
